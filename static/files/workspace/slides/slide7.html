Multi-Head -> Grouped-Query -> Multi-Query

MHA (Original)

One K and V head per query head. Maximum expressivity but large KV cache.

Full flexibility

GQA (2023)

Groups of query heads share K/V. Balance between memory and quality.

Best trade-off

MQA

Single K/V for all queries. Minimal cache but some quality loss.

Fastest inference
